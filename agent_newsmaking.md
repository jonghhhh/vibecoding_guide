# AI와 함께하는 기사 작성 (기초)

### Claude Code의 CLAUDE.md · 스킬 · 서브에이전트를 가장 간단하게 경험하기

> 대상: 코딩 경험이 거의 없는 기자
> 도구: Claude Code (터미널 기반 AI 코딩 에이전트)
> 목표: 자료를 폴더에 넣으면 핵심 정리 → (검색 보완) → 내 문체로 기사 작성 → 사실 검증까지 한 번에. 그 과정에서 Claude Code의 세 기능을 직접 경험한다.
> 핵심 원칙: AI는 보조 기자이며, 최종 판단과 책임은 사람 기자에게 있다.

\---

## 1\. 무엇을 만들고, 무엇을 경험하나

자료 몇 개를 넣고 "기사 만들어줘" 한 마디면 다음 순서로 진행됩니다.

```text
sources/ 의 자료
   │
   ▼ 1. 핵심 정리        → output/key\_facts.md   (메인 세션이 직접)
   │
   ▼ 2. 검색 보완(선택)   → 웹에서 배경·최신 수치 보완 (메인 세션, WebSearch)
   │
   ▼ 3. 기사 작성        → output/article.md     (스킬 + 내 문체 + 뉴스가치)
   │
   ▼ 4. 사실 검증        → output/factcheck.md   (서브에이전트, 독립 검증)
```

경험하는 세 기능은 다음과 같습니다.

* CLAUDE.md: 프로젝트의 규칙·작업 순서·내 문체·뉴스가치를 담아 두면, 매번 자동으로 적용됩니다.
* 스킬(1개): 기사 작성 절차를 정리해 두면, 작성 단계에서 자동으로 그 절차가 적용됩니다.
* 서브에이전트(1개): 검증을 별도의 독립된 문맥에서 수행해, 작성한 내용에 휘둘리지 않고 따로 점검합니다.

핵심 정리와 검색 보완은 별도 파일 없이 메인 세션(기본 Claude)이 직접 합니다. 메인 세션은 파일 읽기(Read)와 웹 검색(WebSearch)을 기본으로 갖고 있어, CLAUDE.md에 적어 둔 순서대로 처리합니다.

\---

## 2\. 폴더 구조 (최대한 간결하게)

```text
agent\_newsmaking/
├── CLAUDE.md          # 규칙 + 작업 순서 + 뉴스가치 + 내 문체
├── my\_style.txt       # 내 기사 샘플 2\~3편 (문체 학습용)
├── sources/           # 원자료 (txt 또는 텍스트형 PDF)
├── output/            # 결과물 (핵심 정리·기사·검증)
└── .claude/
    ├── skills/
    │   └── write-article/SKILL.md   # 스킬 1개 (기사 작성)
    └── agents/
        └── fact-checker.md          # 서브에이전트 1개 (검증)
```

파일은 사실상 CLAUDE.md, my\_style.txt, 스킬 1개, 에이전트 1개 — 이렇게 네 개뿐입니다.

## 3\. 준비

### 3-1. Claude Code 실행

설치가 끝났다면(npm install -g @anthropic-ai/claude-code 후 Anthropic 계정 로그인), 폴더를 만들고 그 안에서 실행합니다.

```bash
mkdir agent\_newsmaking \&\& cd agent\_newsmaking
claude
```

### 3-2. 자료 넣기

sources/ 폴더에 원자료(txt 또는 텍스트형 PDF)를 넣습니다. 여러 개 넣어도 모두 읽습니다.

### 3-3. 내 문체 샘플 정리

my\_style.txt 한 파일에 과거 기사 2\~3편을 구분선으로 나눠 붙입니다. 한 편보다 2\~3편이 좋습니다(리드·문장 길이·인용 처리 같은 패턴이 더 잘 드러납니다). 가능하면 성격이 다른 글을 섞으세요(예: 스트레이트 1편 + 해설/기획 1편).

```text
=== 샘플 기사 1 ===
(기사 전문 붙여넣기)

=== 샘플 기사 2 ===
(기사 전문 붙여넣기)

=== 샘플 기사 3 ===
(기사 전문 붙여넣기)
```

## 4\. 설정 파일 만들기

아래 내용을 그대로 저장합니다. (Claude Code 안에서 "아래 내용으로 CLAUDE.md를 만들어 줘"라고 시켜도 됩니다.)

### 4-1. CLAUDE.md

```markdown
# 뉴스 작성 프로젝트 규칙

## 역할
- 너는 우리 뉴스룸의 보조 기자다. 최종 판단과 책임은 사람 기자에게 있다.
- 모든 답과 기사는 한국어로 쓴다.
- 사실·수치·인용은 지어내지 않는다. 근거가 없으면 "\[확인 필요]"로 표시한다.

## 입력/출력
- 원자료: sources/  (txt, 텍스트형 PDF)
- 내 문체 샘플: my\_style.txt
- 결과물: output/

## 작업 순서 ("기사 만들어줘"라고 하면 이 순서대로 진행한다)
1. sources/의 모든 파일을 읽고 핵심 사실을 정리한다. 각 사실에 출처(파일명·위치)를 단다. → output/key\_facts.md
2. 배경·최신 수치가 더 필요하면 웹에서 검색해 보완하고, 출처 URL을 단다(선택).
3. write-article 스킬로 기사를 쓴다. → output/article.md
4. fact-checker 서브에이전트로 기사를 검증한다. → output/factcheck.md
각 단계 결과는 output/에 파일로 남기고, 단계마다 무엇을 했는지 한 줄로 보고한다.

## 뉴스가치 (기사에서 무엇을 앞세울지)
- 새로움·중요성·사회적 파장·유명성·갈등 등으로 평가해, 가장 가치 높은 사실을 리드와 앵글로 삼는다.
- 가치는 선택과 강조에만 쓰고, 사실 자체는 과장·왜곡하지 않는다.

## 내 문체
- 상세 문체는 my\_style.txt의 샘플을 따른다.
- 리드는 한 문장, 핵심 수치를 앞세운다. 문장은 짧게, 번역투를 피한다.

## 금지
- 출처 없는 단정, 가짜 인용, 수치 왜곡(단위·기간 누락), 낚시성 제목.
```

### 4-2. 스킬 — .claude/skills/write-article/SKILL.md

```markdown
---
name: write-article
description: 정리된 핵심 사실을 바탕으로 내 문체로 기사를 쓸 때 사용한다.
---
# 기사 작성 절차
1. output/key\_facts.md(와 검색 보완 내용)만 근거로 쓴다. 근거 없는 문장은 쓰지 않는다.
2. 시작 전 my\_style.txt를 읽어 리드 방식·문장 길이·어휘·인용 패턴을 파악한다.
3. 뉴스가치(새로움·중요성·사회적 파장·유명성·갈등)로 사실을 평가해, 가장 가치 높은 것을 리드와 앵글로 삼는다. 단 과장·왜곡은 금지.
4. 구조: 리드(핵심 한 문장) → 핵심 사실 → 인용 → 배경 → 전망.
5. 해석은 \[추론]으로 표시한다.
6. output/article.md 로 저장한다(기본 800자).
```

### 4-3. 서브에이전트 — .claude/agents/fact-checker.md

```markdown
---
name: fact-checker
description: 작성된 기사의 사실·수치·인용을 핵심 사실 정리와 원문에 대조해 검증할 때 사용.
tools: Read, Grep, WebSearch, WebFetch
model: opus
---
너는 깐깐한 팩트체크 데스크다. 기사의 검증 가능한 모든 문장을 하나씩 점검한다.
각 문장의 근거를 output/key\_facts.md와 원문 sources/에서 찾아 매칭하고 판정한다.
판정: \[확인] / \[출처 불일치] / \[근거 없음] / \[과장].
인용은 원문과 글자 단위로, 수치는 단위·기간을 대조한다.
문제 항목은 무엇이 틀렸는지와 수정안을 함께 제시한다.
결과는 '문장 / 판정 / 근거 / 수정안' 표로 내고, 통과 못한 항목을 먼저 경고한다.
고치지 말고 지적만 한다.
```

검증자에게는 Edit·Write 권한을 주지 않았습니다. 고치지 않고 지적만 하게 해서, 작성과 검증을 분리합니다.

\---

## 5\. 실행

준비가 끝나면 한 마디면 됩니다.

```text
기사 만들어줘. CLAUDE.md의 작업 순서대로 sources를 읽고 핵심 정리 → 기사 작성 → 검증까지 진행하고, 단계마다 한 줄로 보고해.
```

기능을 하나씩 체감하고 싶을 때는 단계를 끊어서 시키면 됩니다.

```text
sources 폴더의 모든 자료를 읽고 핵심 사실만 정리해서 output/key\_facts.md 로 저장해.
```

```text
방금 정리한 사실로 write-article 스킬을 써서 기사를 써 줘.
```

```text
fact-checker 서브에이전트로 기사를 검증해 줘.
```

큰 작업 전에 Shift+Tab으로 플랜 모드에 들어가면, 어떤 스킬·서브에이전트가 언제 작동하는지 계획으로 먼저 보여 주어 교육용으로 좋습니다.

## 6\. PDF에 대해

텍스트형 PDF(복사·검색이 되는 일반 보도자료·리포트)와 txt는 Claude Code가 그대로 읽습니다. 강의 전에 샘플 PDF 한두 개를 열어 텍스트가 제대로 잡히는지 확인해 두면 안전합니다.

스캔본(이미지 PDF)이나 표·수치가 빽빽한 공시는 그대로는 빠지거나 어긋날 수 있습니다. 이런 자료가 많으면 심화 과정(PDF를 마크다운으로 변환하는 pdf-to-md 단계가 포함된 풀버전)으로 넘어가세요.

\---

## 7\. 잊지 말 것

검색 보완은 Claude Code 내장 WebSearch로 충분합니다. 더 정밀한 한국어 출처나 정확한 수치가 중요해지면, 검색 MCP(Tavily·Brave 등)를 하나만 붙이면 됩니다(여러 개는 오히려 혼란).

윤리 두 가지를 강조합니다. 첫째, AI 보조로 작성했다는 사실을 밝힙니다. 둘째, 최종 인용과 수치는 사람 기자가 1차 출처로 다시 확인합니다. AI는 초안과 검증을 돕는 도구이며, 출고 책임은 사람에게 있습니다.
