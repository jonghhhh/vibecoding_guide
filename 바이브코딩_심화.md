### 슬래시 명령어 & 확장 기능 총정리 — Claude Code · OpenAI Codex · Gemini CLI

> 입문 특강(설치·첫 실습)을 마친 분이 **"같은 말을 더 똑똑하게, 더 자동으로 시키는 법"**을 배우는 단계입니다.
> **핵심 메시지**: 세 도구는 **이름만 다를 뿐 거의 같은 6개 부품**으로 이루어져 있습니다. 하나를 이해하면 나머지는 대응표만 봐도 됩니다.

## 0. 가장 먼저 보는 "개념 대응표"

바이브 코딩 도구의 확장 기능은 **6개 부품**으로 정리됩니다. 세 도구에서 이름이 어떻게 다른지 한눈에 보세요.

|부품|한 줄 설명|**Claude Code**|**OpenAI Codex**|**Gemini CLI**|
|-|-|-|-|-|
|**① 슬래시 명령어**|세션을 조종하는 단축 명령(`/`)|`/help`, `/model`…|`/help`, `/model`…|`/help`, `/model`…|
|**② 메모리 파일**|프로젝트 규칙을 매번 기억|`CLAUDE.md`|`AGENTS.md`|`GEMINI.md`|
|**③ 커스텀 명령어**|나만의 `/명령`을 직접 제작|`.claude/commands/*.md`|`~/.codex/prompts/*.md`|`.gemini/commands/*.toml`|
|**④ 스킬(Skill)**|상황에 맞게 자동 발동하는 전문 능력|`.claude/skills/SKILL.md`|`/skills`(`$`로 호출)|`/skills`·확장|
|**⑤ 서브에이전트**|독립된 두뇌를 가진 보조 일꾼|`.claude/agents/*.md`|서브에이전트 스레드|`/agents`|
|**⑥ 훅(Hook)**|특정 순간 자동 실행되는 스크립트|`settings.json` 훅|`/hooks`|`/hooks`|
|**⑦ MCP**|외부 도구(깃허브·DB 등) 연결|`.mcp.json`|`config.toml`|`settings.json`·확장|

> **비유로 한 번에 이해하기**
> - **메모리 파일** = 신입에게 주는 *업무 매뉴얼* (매일 아침 읽음)
> - **슬래시 명령어** = *리모컨 버튼* (내가 눌러서 조종)
> - **커스텀 명령어** = 내가 만든 *나만의 리모컨 버튼*
> - **스킬** = 상황 보고 알아서 꺼내 쓰는 *전문 매뉴얼*
> - **서브에이전트** = 일을 떼어 맡기는 *별도 직원* (자기 책상=문맥을 따로 씀)
> - **훅** = "이 일 끝나면 자동으로 ○○해" 하는 *자동 알람*
> - **MCP** = 외부 회사와 연결하는 *전용 전화선*

## 1. 슬래시 명령어의 기본기 (3도구 공통)

### 1-1. 슬래시 명령어란?

세션 입력창에 **`/`** 를 치면 뜨는 **조종 명령**입니다. AI에게 보내는 일반 프롬프트와 달리, **세션 자체**(모델 변경·문맥 정리·승인 설정 등)를 다룹니다.

* 입력창에 `/` 를 치면 **명령 목록 팝업**이 뜹니다. 계속 타이핑하면 필터링됩니다.
* 세 도구 모두 `/help` 로 **현재 버전에서 쓸 수 있는 전체 목록**을 볼 수 있습니다. (가장 확실한 방법)

### 1-2. 슬래시와 함께 쓰는 두 기호: `@` 와 `!` (3도구 공통)

|기호|뜻|예시|
|-|-|-|
|`@`|**파일·폴더를 프롬프트에 첨부**|`@src/main.py 이 파일 설명해 줘`|
|`!`|**셸 명령 바로 실행**(AI 거치지 않음)|`!git status`|

> 슬래시(`/`)는 *세션 조종*, 골뱅이(`@`)는 *맥락 첨부*, 느낌표(`!`)는 *터미널 직접 실행*. 이 3개만 알아도 절반은 끝납니다.

## 3. 메모리 파일 — `CLAUDE.md` · `AGENTS.md` · `GEMINI.md`

### 3-1. 왜 가장 중요한가?

이 파일은 AI의 **"헌법"**입니다. **매 세션 자동으로 읽혀서**, 매번 프로젝트 설명을 반복하지 않아도 됩니다. 고급 기능을 건드리기 전에 **이 파일부터 제대로** 써야 효과가 큽니다.

* 세 도구 모두 `/init` 명령으로 **초안을 자동 생성**할 수 있습니다.
* 파일은 **프로젝트 폴더 최상단**에 둡니다. (전역 규칙은 각 도구의 홈 디렉터리에)

### 3-2. 작성 예시 (세 파일 내용은 거의 동일하게 써도 됨)

```markdown
# 프로젝트 개요
이 저장소는 미디어학과 수업용 뉴스 분석 파이썬 프로젝트다.

# 코딩 규칙
- 모든 주석과 설명은 한국어로 작성한다.
- 의존성은 requirements.txt에 명시하고, 불필요한 라이브러리는 추가하지 않는다.
- 함수에는 한 줄 설명을 단다.

# 자주 쓰는 명령
- 테스트:  pytest
- 실행:    python main.py

# 하지 말 것
- .env 파일을 절대 커밋하지 않는다.
- 대규모 리팩터링은 먼저 계획을 보여주고 승인받는다.
```

> **차이점 요약**
> | | 파일명 | 전역(글로벌) 위치 | 초안 생성 |
> |---|---|---|---|
> | Claude | `CLAUDE.md` | `~/.claude/CLAUDE.md` | `/init` |
> | Codex | `AGENTS.md` | `~/.codex/AGENTS.md` | `/init` |
> | Gemini | `GEMINI.md` | `~/.gemini/GEMINI.md` | `/init` |

> **팁**: 규칙은 **짧고 명령형**으로. "되도록 깔끔하게" 같은 모호한 말보다 "함수는 30줄을 넘기지 말 것"처럼 검증 가능한 규칙이 잘 지켜집니다.

## 4. 커스텀 슬래시 명령어 만들기

자주 쓰는 긴 프롬프트를 **나만의 `/명령` 한 단어**로 저장합니다. 예: `/리뷰`, `/한글주석`, `/커밋메시지`.

### 4-1. Claude Code

`.claude/commands/` 폴더에 **`.md` 파일**을 만들면 파일명이 곧 명령이 됩니다.

```bash
mkdir -p .claude/commands
cat > .claude/commands/한글주석.md << 'EOF'
방금 작성한 코드에 초보자가 이해하기 쉬운 한국어 주석을 충분히 달아 줘.
대상 파일: $ARGUMENTS
EOF
```

→ 이제 `/한글주석 src/main.py` 처럼 사용. (`$ARGUMENTS`=전체 인자, `$1 $2`=위치 인자)

### 4-2. OpenAI Codex

`~/.codex/prompts/` 폴더에 **최상위** `.md` 파일을 둡니다. (하위 폴더는 인식 안 함)

```bash
mkdir -p ~/.codex/prompts
cat > ~/.codex/prompts/draftpr.md << 'EOF'
변경 내용을 요약해 PR 설명 초안을 만들어 줘. 제목과 본문을 나눠서.
EOF
```

→ `/prompts:draftpr` 로 호출. (인자는 `$1`, `$2` …)

> 최근 Codex는 **커스텀 프롬프트보다 "스킬"을 권장**하는 방향입니다(5절 참고).

### 4-3. Gemini CLI

`.gemini/commands/` 폴더에 **`.toml` 파일**을 만듭니다.

```bash
mkdir -p .gemini/commands
cat > .gemini/commands/plan.toml << 'EOF'
description = "구현 없이 단계별 계획만 세운다."
prompt = "다음 작업의 단계별 계획만 작성해. 코드는 쓰지 마: {{args}}"
EOF
```

→ `/commands reload` 후 `/plan 인증 모듈 리팩터링` 사용.

* 하위 폴더로 **네임스페이스** 가능: `.gemini/commands/git/commit.toml` → `/git:commit`
* `{{args}}` 자리에 사용자가 넘긴 인자가 들어갑니다.

## 5. 스킬(Skills) — 상황 보고 자동 발동하는 전문 능력

### 5-1. 커스텀 명령어와 무엇이 다른가?

* **커스텀 명령어** = *내가* `/명령`으로 **직접** 부른다.
* **스킬** = AI가 **상황(맥락)을 보고 알아서** 꺼내 쓴다(원하면 수동 호출도 가능). 예: "엑셀 파일이 보이면 자동으로 엑셀 처리 능력을 로드".

### 5-2. Claude Code (가장 정돈됨)

`.claude/skills/<이름>/SKILL.md` 폴더 구조로 만듭니다. 2026년 기준 **슬래시 명령어와 스킬이 통합**되어, 모든 스킬은 `/스킬이름` 으로도 부를 수 있습니다.

```markdown
---
name: news-cleaner
description: 뉴스 CSV의 중복·결측을 정리할 때 사용한다.
# 프론트매터로 자동 발동 여부, 메뉴 노출, 서브에이전트 실행을 제어
---
# 뉴스 CSV 정리 절차
1. 중복 기사 제거 (제목+날짜 기준)
2. 빈 본문 행 표시
...
```

* 인자 치환: `$ARGUMENTS`, `$1 $2`, 스킬 폴더 경로 `${CLAUDE_SKILL_DIR}`.

### 5-3. Codex / Gemini

* **Codex**: `/skills` 로 보기, `$스킬이름` 으로 직접 호출. (커스텀 프롬프트의 후속 개념)
* **Gemini**: `/skills`·확장(Extension) 형태로 제공(버전에 따라 다름). `/skills` 로 확인.

## 6. 서브에이전트(Subagents) — 일을 떼어 맡기는 독립 일꾼

### 6-1. 왜 쓰는가?

복잡한 작업을 **별도의 두뇌(독립 문맥)**에게 맡기면, ① 본 대화가 깔끔하게 유지되고 ② 토큰을 아끼며 ③ 역할별 전문화가 됩니다. 예: "코드 탐색 담당", "리뷰 담당".

### 6-2. Claude Code

`.claude/agents/` 에 정의하거나 `/agents` 로 관리합니다. 내장 유형:

* **Explore**: 읽기 전용 코드베이스 탐색
* **Plan**: 실행 없이 계획 수립
* **general-purpose**: 범용

```markdown
---
name: reviewer
description: 보안·테스트 관점으로 변경 코드를 검토한다.
tools: Read, Grep, Bash
model: haiku        # 가벼운 작업은 저렴한 모델로
---
너는 까다로운 코드 리뷰어다. 보안 취약점과 누락된 테스트를 우선 지적해라.
```

### 6-3. Codex / Gemini

* **Codex**: 서브에이전트 스레드로 분기, 리뷰를 **guardian 서브에이전트**로 라우팅 가능. `/agents`.
* **Gemini**: `/agents list` 로 서브에이전트 확인·관리.

> **선택 기준**: 작업이 **독립된 문맥**을 필요로 하면 서브에이전트, 단순 반복이면 커스텀 명령어/스킬.

## 7. 훅(Hooks) — 특정 순간 자동 실행

### 7-1. 개념

"**도구 실행 전/후**"나 "**프롬프트 제출 시**" 같은 **이벤트**에 **내 스크립트를 자동 실행**합니다. 예: 코드 저장 후 자동 포맷팅, 비밀키가 들어가면 차단, 작업 끝나면 알림.

### 7-2. Claude Code (가장 세밀)

`settings.json` 에 이벤트별로 등록합니다. 대표 이벤트: `PreToolUse`(도구 실행 전), `PostToolUse`(후), `UserPromptSubmit`(프롬프트 제출 시), `Notification`, `Stop` 등.

```jsonc
// .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{ "type": "command", "command": "black ." }]
      }
    ]
  }
}
```

→ 파일을 수정/작성할 때마다 자동으로 `black`(파이썬 포매터) 실행.

### 7-3. Codex / Gemini

* **Codex**: `/hooks` 관리. 예) `userpromptsubmit` 훅으로 프롬프트를 **차단하거나 보강**한 뒤 실행.
* **Gemini**: `/hooks` 로 관리.

> 훅은 **자동으로 명령을 실행**하므로, 신뢰할 수 있는 스크립트만 등록하세요.

## 8. MCP — 외부 도구 연결 (Model Context Protocol)

### 8-1. 개념

MCP는 AI를 **깃허브·노션·DB·브라우저·구글 워크스페이스** 같은 외부 도구에 연결하는 **표준 규격(전용 전화선)**입니다. 한 번 연결하면 세 도구 모두 동일하게 "외부 도구를 호출"할 수 있습니다. (세 도구 다 `/mcp` 로 상태 확인)

### 8-2. 도구별 설정 위치

|도구|설정 방법|
|-|-|
|**Claude Code**|프로젝트의 `.mcp.json`, 또는 `claude mcp add ...` → `/mcp` 로 관리|
|**OpenAI Codex**|`~/.codex/config.toml` 의 `mcp_servers`, `codex mcp` → `/mcp`|
|**Gemini CLI**|`settings.json`(또는 확장)에 서버 등록 → `/mcp`|

### 8-3. 예시 — 깃허브 MCP 서버 (Claude 기준)

```jsonc
// .mcp.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "여기에_토큰" }
    }
  }
}
```

→ 이후 `/mcp` 로 연결 확인 후 "내 깃허브 이슈 목록 가져와" 처럼 자연어로 요청.

> 토큰·키는 **항상 환경변수/`.env`**로. 설정 파일을 깃허브에 올릴 땐 키가 없는지 확인.

### 8-4. 플러그인 / 확장 (여러 부품을 한 번에 묶어 배포)

* **Claude**: **플러그인** = 스킬+서브에이전트+명령어+훅+MCP를 한 묶음으로 설치(`/plugin`).
* **Codex**: `/plugins`, `/apps`.
* **Gemini**: **확장(Extension)** = 커스텀 명령어+MCP+`GEMINI.md`를 한 패키지로(`gemini extensions`, `/extensions`).

## 9. 언제 무엇을 쓸까? — 의사결정 가이드

```
무엇을 자동화/확장하고 싶은가?
├── 프로젝트의 '항상 지켜야 할 규칙'      → ② 메모리 파일 (CLAUDE/AGENTS/GEMINI.md)
├── 내가 직접 부르는 반복 프롬프트        → ③ 커스텀 슬래시 명령어
├── 상황 보고 '알아서' 적용될 전문 지식   → ④ 스킬(Skill)
├── 독립된 문맥이 필요한 큰 하위 작업     → ⑤ 서브에이전트
├── 특정 순간 '자동으로' 실행할 동작      → ⑥ 훅(Hook)
└── 외부 서비스(깃허브·DB·노션) 연결     → ⑦ MCP
```

|원하는 것|답|핵심 특징|
|-|-|-|
|매번 같은 규칙 적용|메모리 파일|세션마다 자동 로드|
|내가 타이밍을 정해 실행|커스텀 명령어|`/명령`으로 수동 호출|
|AI가 알아서 판단해 적용|스킬|맥락 기반 자동 발동|
|깨끗한 별도 작업 공간|서브에이전트|독립 문맥·도구·권한|
|이벤트에 반응하는 자동화|훅|트리거 기반 실행|
|외부 도구 호출|MCP|표준 연결 규격|

## 10. 안전 수칙 & 자주 막히는 곳

**안전 수칙**

1. **`--yolo` / `danger-full-access` 는 격리 환경에서만.** 승인·샌드박스를 모두 끕니다.
2. **비밀키는 절대 코드·설정 파일에 직접 쓰지 않기.** `.env` + `.gitignore`.
3. **큰 작업은 플랜 모드로 먼저 계획 확인** (`/plan`, Claude는 `Shift+Tab`).
4. **훅·MCP·플러그인은 신뢰할 수 있는 출처만** 등록(자동 실행되기 때문).

**자주 막히는 곳**

|증상|해결|
|-|-|
|슬래시 명령이 안 보임|버전 차이 → `/help`로 현재 목록 확인, 도구 업데이트|
|커스텀 명령이 안 뜸|Gemini는 `/commands reload`, Claude/Codex는 파일 위치·확장자 확인|
|메모리 파일이 무시됨|파일이 **프로젝트 최상단**에 있는지, 파일명이 정확한지 확인|
|MCP 도구가 안 보임|`/mcp`(또는 `/mcp verbose`)로 연결·토큰 진단|
|Gemini가 6/18 이후 안 됨|**Antigravity CLI**로 이전|
